# Impeccable로 ETF 큐레이션 웹서비스 디자인하기

> AI 코딩 어시스턴트에게 "디자인 감각"을 심어주는 도구, Impeccable을 활용해 ETF 큐레이션 서비스의 UI를 설계하는 워크샵 기록

---

## 왜 이 워크샵이 필요한가

AI 코딩 어시스턴트에게 "ETF 큐레이션 페이지 만들어줘"라고 요청하면, 십중팔구 이런 결과물이 나온다:

- Inter 폰트에 보라색 그라데이션 히어로 섹션
- 회색 배경 위에 둥근 모서리 카드 그리드
- "Start Your Investment Journey" 같은 공허한 헤드라인
- 어디서 많이 본 것 같은, 누가 봐도 AI가 만든 느낌의 레이아웃

이른바 **"AI Slop"** — LLM이 학습한 수만 개의 웹 템플릿에서 가장 평균적인 패턴을 뽑아낸 결과다. 금융 서비스처럼 신뢰와 전문성이 생명인 도메인에서 이런 제네릭한 디자인은 치명적이다.

**Impeccable**은 이 문제를 정면으로 해결한다. CSS 프레임워크가 아니라, AI 어시스턴트에게 전문 디자이너 수준의 판단력을 부여하는 **디자인 스킬 시스템**이다.

---

## Part 1: Impeccable 이해하기

### Impeccable은 무엇인가

[Impeccable](https://github.com/pbakaus/impeccable)은 Paul Bakaus가 만든 오픈소스 프로젝트(Apache 2.0)로, AI 코딩 어시스턴트(Claude Code, Cursor, Gemini CLI 등)에 설치하는 **디자인 지식 레이어**다.

핵심 개념은 간단하다:

> AI가 코드를 *작성하는 방법*은 이미 알고 있다.
> 부족한 건 *무엇이 좋은 디자인인지* 판단하는 능력이다.
> Impeccable은 그 판단력을 제공한다.

### 작동 방식

Impeccable은 런타임 CSS를 전혀 추가하지 않는다. 대신 구조화된 마크다운 스킬 파일을 AI 에디터의 설정 디렉토리에 설치하여, 코드 생성 시점에 디자인 의사결정을 가이드한다.

```
┌─────────────────────────────────────────┐
│  사용자: "ETF 목록 페이지 만들어줘"       │
│                  ↓                       │
│  ┌─────────────────────────────────┐     │
│  │  Impeccable 디자인 지식 레이어    │     │
│  │  • 타이포그래피 시스템            │     │
│  │  • 색상 & 대비 규칙              │     │
│  │  • 공간 설계 원칙                │     │
│  │  • 안티패턴 감지                 │     │
│  │  • 도메인 컨텍스트 (.impeccable) │     │
│  └─────────────────────────────────┘     │
│                  ↓                       │
│  AI가 디자인 판단을 반영한 코드 생성      │
└─────────────────────────────────────────┘
```

### 7개 디자인 도메인 모듈

| 모듈 | 다루는 영역 | ETF 서비스 적용 포인트 |
|------|------------|---------------------|
| **Typography** | 폰트 시스템, 모듈러 스케일, OpenType | 수익률 숫자의 가독성, 테이블 정렬 |
| **Color & Contrast** | OKLCH 색공간, 접근성, 다크모드 | 상승/하락 색상, 리스크 등급 표현 |
| **Spatial Design** | 스페이싱 시스템, 그리드, 시각 위계 | 카드 간격, 정보 밀도 조절 |
| **Motion Design** | 이징 커브, 스태거링, reduced-motion | 차트 애니메이션, 로딩 전환 |
| **Interaction Design** | 폼 패턴, 포커스 상태, 로딩 인디케이터 | 필터 UI, 검색, 비교 기능 |
| **Responsive Design** | 모바일 퍼스트, 컨테이너 쿼리 | 모바일 포트폴리오 뷰 |
| **UX Writing** | 버튼 레이블, 에러 메시지, 빈 상태 | 투자 용어 안내, 빈 포트폴리오 상태 |

### `/teach-impeccable` 실행 — 디자인 컨텍스트 확립

Impeccable의 첫 단계는 항상 컨텍스트 수집이다. `/teach-impeccable`을 실행하면 대화형으로 다음을 파악한다:

- **타깃 사용자**: 30-50대, 투자 경험 1-5년, ETF를 통한 분산투자에 관심
- **브랜드 성격**: 신뢰할 수 있는(Trustworthy), 명료한(Clear), 전문적이되 위압적이지 않은(Approachable Expert)
- **미적 방향**: 깔끔하고 데이터 중심, 불필요한 장식 배제, 숫자가 주인공
- **경쟁 서비스 참고**: 미래에셋 MAP, 삼성증권 mPOP, 토스증권

이 정보가 `.impeccable.md` 파일로 저장되어, 이후 모든 디자인 작업의 기준점이 된다.

```markdown
# .impeccable.md (생성 예시)

## Audience
- Korean retail investors, 30-50 age range
- Moderate investment experience (1-5 years)
- Seeking curated ETF portfolios, not self-directed stock picking

## Brand Personality
- Trustworthy & institutional, but not cold
- Data-driven clarity over decorative flourish
- Numbers are the hero, UI is the stage

## Aesthetic Direction
- Clean data-forward layouts inspired by Bloomberg Terminal meets consumer app
- Muted palette with intentional accent colors for financial signals
- Tabular number fonts, generous whitespace, strong typographic hierarchy

## Anti-Patterns to Avoid
- Generic "fintech purple gradient" hero sections
- Overly playful illustrations that undermine credibility
- Hiding important data behind unnecessary interactions
- Dark patterns in portfolio recommendations
```

## Part 3: ETF 큐레이션 서비스 설계

### 서비스 구조 정의

ETF 큐레이션 서비스의 핵심 화면을 먼저 정의한다:

```
ETF 큐레이션 서비스
├── 홈 / 대시보드
│   ├── 오늘의 추천 포트폴리오
│   ├── 시장 요약 (주요 지수)
│   └── 인기 ETF 랭킹
├── 탐색 (Explore)
│   ├── 테마별 ETF (AI, 반도체, 배당, ESG ...)
│   ├── 섹터별 ETF
│   └── 글로벌 / 국내 필터
├── ETF 상세
│   ├── 성과 차트 (1M, 3M, 6M, 1Y, 3Y)
│   ├── 구성 종목 Top 10
│   ├── 비용 비율 & 배당 이력
│   └── 유사 ETF 비교
├── 큐레이션 (핵심 기능)
│   ├── 투자 성향 진단
│   ├── 맞춤 포트폴리오 추천
│   ├── 리밸런싱 알림
│   └── 백테스트 시뮬레이션
└── 마이 포트폴리오
    ├── 보유 현황
    ├── 수익률 추이
    └── 리밸런싱 이력
```

---

### Step 1: 홈 대시보드 초안 설계

AI에게 첫 화면을 요청하되, Impeccable의 디자인 지식이 반영되도록 한다.

**프롬프트 예시:**
```
ETF 큐레이션 서비스의 홈 대시보드를 만들어줘.
- 오늘의 추천 포트폴리오 3개 (보수적/균형/공격적)
- 주요 지수 현황 (KOSPI, KOSDAQ, S&P500, NASDAQ)
- 테마별 인기 ETF Top 5
React + Tailwind CSS 사용
```

**Impeccable이 개입하는 지점들:**

Impeccable이 없었다면 나왔을 코드 vs 있을 때의 차이:

| 요소 | Without Impeccable | With Impeccable |
|------|-------------------|----------------|
| 폰트 | `font-family: Inter` 일률적 적용 | 본문은 Pretendard, 숫자는 `font-variant-numeric: tabular-nums` 적용 |
| 색상 | `text-green-500` / `text-red-500` | OKLCH 기반 상승(teal-600)/하락(rose-600), 색맹 접근성 고려 |
| 카드 | `rounded-lg shadow-md` 균일 카드 | 위계에 따른 차별화 — 추천 포트폴리오는 강조, 지수는 인라인 |
| 레이아웃 | 3열 균등 그리드 | 시선 흐름을 고려한 비대칭 레이아웃, F-패턴 적용 |
| 숫자 표현 | `+2.34%` 텍스트만 | 방향성 아이콘 + 색상 + 볼드 조합, 시각적 즉시 인지 가능 |

### Step 2: `/critique` — UX 관점 리뷰

초안이 나오면 `/critique`를 실행한다. 이 커맨드는 코드를 수정하지 않고, UX 관점의 리뷰 리포트만 생성한다.

**리뷰에서 잡아낼 수 있는 문제들:**

```markdown
## /critique 리포트 (예시)

### 시각 위계 (Visual Hierarchy)
⚠️ 추천 포트폴리오 3개가 동일한 시각적 무게를 가짐
→ 사용자 성향에 맞는 포트폴리오를 시각적으로 강조해야 함

### 정보 과부하 (Information Overload)
⚠️ 대시보드에 지수 4개 + 포트폴리오 3개 + ETF 5개 = 동시에 12개 데이터 블록
→ 프로그레시브 디스클로저 적용 권장

### 행동 유도 (Call to Action)
⚠️ "자세히 보기" 버튼이 모든 카드에 동일한 스타일로 반복
→ 주요 CTA(투자 성향 진단)와 보조 CTA를 구분해야 함

### 빈 상태 (Empty State)
❌ 신규 사용자의 "마이 포트폴리오" 빈 상태 미설계
→ 투자 성향 진단으로 유도하는 온보딩 필요
```

### Step 3: `/typeset` — 금융 데이터의 타이포그래피

금융 서비스에서 타이포그래피는 미적 요소가 아니라 **기능적 요소**다. 숫자의 가독성이 곧 사용성이다.

**`/typeset`이 적용하는 규칙들:**

```css
/* 수익률, 가격 등 숫자 데이터 */
.financial-number {
  font-variant-numeric: tabular-nums lining-nums;
  letter-spacing: -0.01em;
  font-feature-settings: "tnum" 1;
}

/* 퍼센트 변동폭 — 부호와 숫자의 정렬 */
.change-value {
  font-variant-numeric: tabular-nums;
  text-align: right;
  min-width: 5ch; /* ±00.00% 폭 확보 */
}

/* 모듈러 스케일 — 위계를 만드는 크기 시스템 */
:root {
  --text-xs: 0.75rem;   /* 12px — 보조 레이블 */
  --text-sm: 0.875rem;  /* 14px — 테이블 본문 */
  --text-base: 1rem;    /* 16px — 본문 */
  --text-lg: 1.125rem;  /* 18px — 섹션 제목 */
  --text-xl: 1.5rem;    /* 24px — 페이지 제목 */
  --text-2xl: 2rem;     /* 32px — 대시보드 핵심 숫자 */
}
```

**Before/After:**

```
Before:  KOSPI 2,847.23  +1.2%  (모든 텍스트 동일 크기, 동일 무게)

After:   KOSPI          ← text-sm, font-medium, muted color
         2,847.23       ← text-2xl, font-bold, tabular-nums
         +1.2%          ← text-base, font-semibold, teal-600
```

### Step 4: `/colorize` — 금융 신호를 위한 색상 시스템

ETF 서비스에서 색상은 장식이 아니라 **정보 전달 수단**이다.

**`/colorize`가 구축하는 색상 체계:**

```css
:root {
  /* 시장 신호 — OKLCH 기반으로 일관된 시각적 무게 */
  --signal-up: oklch(0.55 0.15 170);      /* Teal — 상승 */
  --signal-down: oklch(0.55 0.15 15);     /* Rose — 하락 */
  --signal-neutral: oklch(0.55 0.02 250); /* Slate — 보합 */

  /* 리스크 등급 */
  --risk-low: oklch(0.65 0.12 155);       /* Green — 안정 */
  --risk-moderate: oklch(0.65 0.12 85);   /* Amber — 보통 */
  --risk-high: oklch(0.60 0.15 25);       /* Red-Orange — 위험 */

  /* 브랜드 팔레트 — 신뢰감 있는 블루 계열 */
  --brand-primary: oklch(0.50 0.12 250);
  --brand-surface: oklch(0.98 0.005 250);
  --brand-muted: oklch(0.70 0.02 250);

  /* 다크모드 자동 대응 */
  @media (prefers-color-scheme: dark) {
    --signal-up: oklch(0.70 0.15 170);
    --signal-down: oklch(0.70 0.15 15);
    --brand-surface: oklch(0.15 0.01 250);
  }
}
```

**접근성 고려사항:**
- 색상만으로 정보를 전달하지 않음 → 아이콘(▲▼)과 텍스트 부호(+/-)를 병행
- 색맹 사용자를 위해 상승/하락에 밝기(Lightness) 차이도 확보
- WCAG AA 이상 대비율 보장

### Step 5: `/arrange` — 정보 밀도와 레이아웃

ETF 서비스의 난제는 **많은 데이터를 압도적이지 않게 보여주는 것**이다.

**`/arrange`가 해결하는 레이아웃 문제들:**

#### 대시보드 레이아웃 — F-패턴 적용

```
┌──────────────────────────────────────────────┐
│  시장 요약 바 (KOSPI ▲ | KOSDAQ ▼ | S&P ▲)  │  ← 가장 먼저 스캔
├──────────────────────┬───────────────────────┤
│                      │                       │
│  맞춤 추천 포트폴리오  │  투자 성향 진단       │  ← 핵심 CTA
│  (넓은 영역, 강조)    │  Quick Start          │
│                      │                       │
├──────────────────────┴───────────────────────┤
│                                              │
│  테마별 인기 ETF                              │  ← 탐색 유도
│  [AI·반도체] [배당] [ESG] [글로벌] ...         │
│                                              │
├──────────────────────────────────────────────┤
│  최근 본 ETF / 관심 ETF                       │  ← 개인화
└──────────────────────────────────────────────┘
```

#### ETF 비교 테이블 — 스캐너블 디자인

```
┌─────────────┬────────┬────────┬────────┐
│ ETF         │ 1Y 수익│ 비용비율│ 배당률  │
├─────────────┼────────┼────────┼────────┤
│ TIGER 200   │ +12.3% │ 0.05%  │ 1.8%   │  ← 행 간 구분선 대신
│             │        │        │        │     미묘한 배경색 교차
│ KODEX 200   │ +11.8% │ 0.04%  │ 1.9%   │
│             │        │        │        │     열 정렬: 숫자는 우측
│ KBSTAR 200  │ +12.1% │ 0.02%  │ 1.7%   │     텍스트는 좌측
└─────────────┴────────┴────────┴────────┘
  최저 비용 표시: 셀 배경 하이라이트 (안전색)
  최고 수익 표시: 셀 텍스트 볼드 + 강조색
```

**Impeccable의 Spatial Design 원칙:**
- **8px 기준 스페이싱 시스템** — 4, 8, 12, 16, 24, 32, 48, 64px
- **컨테이너 쿼리 활용** — 카드 내부 레이아웃이 카드 크기에 반응
- **시각적 그루핑** — 근접성(proximity)으로 관련 데이터를 묶음, 선(border) 남용 금지

---

### Step 6: `/audit` — 품질 감사

개발이 어느 정도 진행되면 `/audit`으로 전방위 품질 점검을 한다.

**`/audit` 체크리스트:**

```markdown
## 접근성 (Accessibility)
✅ 색상 대비율 WCAG AA 충족
✅ 키보드 내비게이션 완전 지원
⚠️ 차트 컴포넌트에 aria-label 누락
❌ 스크린 리더로 포트폴리오 비율 파이차트 인식 불가 → 대체 텍스트 필요

## 성능 (Performance)
✅ 초기 로딩 시 위 영역(Above the fold) 데이터만 fetch
⚠️ ETF 목록 무한스크롤에 가상화(virtualization) 미적용
✅ 이미지 lazy loading 적용

## 반응형 (Responsive)
✅ 모바일에서 테이블 → 카드 전환
⚠️ 태블릿(768-1024px) 구간에서 비교 테이블 수평 스크롤 필요
✅ 터치 타겟 최소 44px 충족

## 다크모드 (Dark Mode)
✅ 시스템 설정 자동 감지
⚠️ 차트 그리드 라인 다크모드에서 시인성 부족
✅ 상승/하락 색상 다크모드 대응 완료
```

---

### Step 7: `/polish` — 최종 다듬기

출시 전 마지막 단계. `/polish`는 12개 차원을 순회하며 미세 조정한다.

**Polishing 예시:**

1. **정렬 일관성** — 헤더 내비게이션과 본문 콘텐츠의 좌측 정렬선 통일
2. **호버 상태 정교화** — ETF 카드 호버 시 미묘한 elevation 변화 + 테두리 색상 전환
3. **트랜지션 일관성** — 모든 인터랙티브 요소에 `transition: 150ms ease-out` 통일
4. **로딩 스켈레톤** — 숫자 데이터 로딩 시 실제 데이터 폭에 맞춘 스켈레톤 표시
5. **빈 상태 마이크로카피** — "아직 관심 ETF가 없어요. 테마별 탐색에서 시작해보세요."
6. **숫자 포맷 일관성** — 모든 금액에 천단위 구분자, 수익률은 소수점 2자리 통일

## Part 5: Impeccable 활용 팁 & 안티패턴

### 금융 UI에서 피해야 할 "AI Slop" 패턴

Impeccable은 다음 안티패턴을 명시적으로 감지하고 회피한다:

| 안티패턴 | 왜 문제인가 | Impeccable의 대안 |
|---------|------------|------------------|
| 보라색 그라데이션 히어로 | 핀테크 클리셰, 신뢰 저하 | 데이터가 곧 히어로 — 시장 현황을 최상단에 |
| 모든 것이 카드 | 위계 없는 균일한 정보 블록 | 중요도에 따른 시각적 차별화 |
| "AI가 추천합니다" 배지 | 과도한 기술 마케팅, 불안 유발 | 추천 근거(데이터)를 투명하게 제시 |
| 3D 코인 / 로켓 일러스트 | 투기적 이미지, 전문성 훼손 | 실제 데이터 시각화가 가장 강력한 그래픽 |
| 카드 안의 카드 | 불필요한 중첩, 공간 낭비 | 플랫한 위계와 의미 있는 구분 |
| 동일한 CTA 버튼 반복 | 사용자 결정 마비 | 주요 CTA 1개 + 보조 액션 구분 |

### 커맨드 체이닝 전략

워크샵에서 효과적인 커맨드 순서:

```
Phase 1 — 기초 설계
/teach-impeccable → 컨텍스트 확립
(코드 작성)
/critique → 구조적 문제 파악

Phase 2 — 세부 다듬기
/typeset → 숫자와 텍스트 정리
/colorize → 금융 신호 색상 체계
/arrange → 레이아웃 최적화

Phase 3 — 품질 확보
/audit → 전방위 감사
/harden → 에러 처리, 엣지케이스
/polish → 최종 마무리
```

### Impeccable이 결정하지 않는 것

Impeccable은 만능이 아니다. 다음은 여전히 인간의 판단이 필요한 영역:

- **비즈니스 로직** — 어떤 ETF를 추천할지는 알고리즘과 금융 전문가의 영역
- **규제 준수** — 투자 권유 문구의 법적 적합성 (금투법, 적합성 원칙)
- **브랜드 아이덴티티** — 로고, 서비스명, 핵심 브랜드 컬러의 최종 결정
- **사용자 리서치** — 실제 사용자 인터뷰와 사용성 테스트 결과 반영
- **퍼포먼스 최적화** — 실제 API 응답 시간과 데이터 구조에 기반한 최적화

## Part 6: 워크샵 결과물 정리

### 산출물 체크리스트

```
□ .impeccable.md — 프로젝트 디자인 컨텍스트 파일
□ 디자인 토큰 정의 (색상, 타이포그래피, 스페이싱)
□ 핵심 화면 5종 구현
  □ 홈 대시보드
  □ ETF 탐색 (테마/섹터)
  □ ETF 상세
  □ 투자 성향 진단
  □ 맞춤 포트폴리오 추천 결과
□ /audit 리포트 통과
□ 반응형 대응 (모바일, 태블릿, 데스크톱)
□ 다크모드 대응
□ 접근성 기본 충족 (WCAG AA)
```

### 배운 점

1. **AI 어시스턴트의 디자인 품질은 입력의 품질에 비례한다** — Impeccable은 그 입력을 체계화하는 도구
2. **금융 UI는 "예쁜 것"보다 "정확한 것"이 먼저다** — tabular-nums 하나가 보라색 그라데이션보다 중요
3. **슬래시 커맨드는 디자인 리뷰 프로세스를 대체할 수 있다** — `/critique` → `/audit` → `/polish` 사이클
4. **컨텍스트 파일(.impeccable.md)이 핵심이다** — 이 파일 없이는 여전히 제네릭한 결과가 나온다
5. **Impeccable은 CSS 프레임워크가 아니다** — Tailwind든 vanilla CSS든, 어떤 구현 방식과도 함께 작동한다
